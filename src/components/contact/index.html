Usage Guidelines

  • • Use clear, descriptive labels for all contact form fields
  • • Include validation feedback for better user experience
  • • Provide multiple contact methods (form, phone, email, address)
  • • Use interactive maps when location is important
  • • Include business hours and contact information prominently
  • • Ensure forms are accessible and keyboard navigable

Accessibility Features

  • • Proper form labels and field associations
  • • ARIA attributes for form validation and map descriptions
  • • Keyboard navigation support throughout contact sections
  • • Screen reader compatible map alternatives
  • • High contrast focus indicators
  • • Logical tab order and semantic HTML structure
