Usage Guidelines

  • • Use linear progress for long processes with known completion times
  • • Use circular progress for compact spaces or detailed status displays
  • • Use indeterminate progress when completion time is unknown
  • • Include percentage labels for clarity when space allows
  • • Use semantic colors to indicate progress status (success, warning, error)

Accessibility Features

  • • ARIA role="progressbar" for screen reader compatibility
  • • aria-valuenow, aria-valuemin, aria-valuemax attributes
  • • High contrast colors meet WCAG accessibility standards
  • • Reduced motion support for users with vestibular disorders
  • • Clear visual progression indicators for all users
