Usage Guidelines

  • • Use checkboxes for binary choices that can be toggled independently
  • • Group related checkboxes using fieldset and legend elements
  • • Use indeterminate state for parent checkboxes in hierarchical selections
  • • Provide clear, descriptive labels for each checkbox option
  • • Ensure adequate spacing between checkbox options for touch interfaces

Accessibility Features

  • • Proper form labels and fieldset grouping for screen readers
  • • Keyboard navigation support (Tab, Space for toggle)
  • • High contrast focus indicators with visible outlines
  • • Clear visual distinction between checked, unchecked, and disabled states
  • • ARIA attributes for complex checkbox group relationships
