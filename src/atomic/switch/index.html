Usage Guidelines

  • • Use switches for binary on/off settings that take effect immediately
  • • Include clear labels that describe what the switch controls
  • • Position switches consistently (usually on the right side of the label)
  • • Use loading state when switch changes trigger network requests
  • • Consider disabled state for features not available to current user

Accessibility Features

  • • Proper role="switch" and aria-checked attributes for screen readers
  • • Keyboard navigation support with Space and Enter keys
  • • High contrast focus indicators that meet WCAG AA standards
  • • Clear visual distinction between on/off states
  • • Smooth animations that respect user motion preferences
